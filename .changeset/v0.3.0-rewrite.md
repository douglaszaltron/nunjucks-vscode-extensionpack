## v0.3.0 — Clean Architecture Rewrite

### Architecture
- Separated formatting logic into `formatter.ts` (zero VS Code dependency)
- Simplified `extension.ts` to thin VS Code wrapper
- Config caching with `onDidChangeConfiguration` listener

### Idempotent Formatting
- Preprocessor only separates inline Nunjucks tags (no newline between `>` and `{%`)
- Formatting twice produces identical output — stable on repeated saves

### Nunjucks (per Mozilla docs)
- All block tags: if/elif/else/for/block/macro/filter/raw/verbatim/call
- asyncEach/endeach, asyncAll/endall, set/endset block assignment
- Whitespace control: {%- -%}, {{- -}}
- Inline Nunjucks inside attributes left untouched

### AlpineJS
- Directive highlighting: x-*, :*, @*
- Snippets for all directives and magic properties

### Simplified Configuration
- Reduced from 11 to 6 settings with hardcoded sensible defaults
- Default wrapAttributes: "force-expand-multiline"
- Default wrapLineLength: 0 (unlimited)
- Default maxPreserveNewlines: 1

### Testing
- 32 tests with plain mocha (no @vscode/test-electron needed)
- Runs in 23ms

### Cleanup
- Merged njk.json + njk-html.json into single grammar file
- Removed opinionated component pattern snippets
- File extensions: .njk and .nunjucks only (official standard)
- Professional English throughout

### Standards
- TypeScript strict mode, ES2022 target
- esModuleInterop, skipLibCheck, forceConsistentCasingInFileNames
